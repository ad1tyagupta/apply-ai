# Scan Mode

Use `portals.yml` as the scanner input.

That file should be generated from:

- user geography
- priority companies
- custom priority companies
- preferred role keywords
- excluded keywords
- user limitations

Selected companies are not a hard limit. Use deterministic scanners where available, then use generated market-wide queries and backlog items for broader Codex-assisted discovery.

Before scoring in bulk, collect at least 10 relevant live jobs when available, verify the links, and ask the user to open a few examples to confirm the set looks right.
